# Skill 变更日志

## v2.1.0 - 2026-04-03

### 新增内容深度、图表规范、章节衔接规则

根据用户要求，新增三大核心规则：

1. **读者定位与内容深度**
   - 明确目标读者：中高级软件工程师（5-10年经验）
   - 深入浅出标准：原理有推导、细节给足、不浮于表面
   - 仔细详尽要求：关键概念、工具使用、案例都有完整内容

2. **图表规范**
   - 优先使用 Mermaid（流程图、时序图）
   - 复杂图形使用 SVG
   - 文件位置规范：`assets/diagrams/章节号-图表名.svg`
   - 每个图表必须有文字说明
   - 禁止 ASCII 图表、低分辨率截图

3. **章节衔接**
   - 绝不跳跃：新概念必须有铺垫
   - 承上启下：每章开头回顾上一章
   - 渐进式：难度逐步提升
   - 前后呼应：引用时给出具体章节号

### 新增文档

- `writing-rules.md` 新增第1-3章（读者定位、图表规范、章节衔接）
- 详细规则包括：要求说明、正反示例、检查清单

## v2.0.0 - 2026-04-03

### 重大变更

1. **重构为三层结构**
   - `skill.md` - 精简版，只保留核心规则和强制自检流程
   - `writing-rules.md` - 详细规则，包括禁用词清单、结构要求、改写示例索引
   - `examples/` - 改写示例库，真实案例展示

2. **增加强制自检机制**
   - 每次输出后必须执行自检
   - 自检包括：禁用词扫描、结构比例检查、核心要素验证
   - 必须输出自检报告，格式固定
   - 不通过时必须指出问题并给出修改建议

3. **优化触发词**
   - 旧版："写书"、"书籍大纲"（太泛，易误触发）
   - 新版："人机协作编程"、"写第X章"、"审核第X章"（精准）

4. **分离 Git 流程**
   - Git 协作流程移到项目根目录 `GIT_WORKFLOW.md`
   - Skill 专注写作规则，避免喧宾夺主

5. **增加改写示例库**
   - `bad-to-good-01.md` - 开篇段落改写
   - `bad-to-good-02.md` - 理论部分改写
   - `bad-to-good-03.md` - 实践工具改写
   - 每个示例包含：原文、问题分析、改写版本、改写要点

### 问题修复

- 修复：规则无 enforcement 机制 → 增加强制自检
- 修复：缺少改写示例 → 增加示例库
- 修复：Git 流程喧宾夺主 → 分离到项目文档
- 修复：触发词太泛 → 精准匹配
- 修复：无版本迭代机制 → 增加 changelog
