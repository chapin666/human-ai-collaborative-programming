# 贡献指南

感谢你对《人机协作编程》的兴趣！我们欢迎各种形式的贡献。

## 如何贡献

### 1. 提交问题或建议

如果你发现：
- 内容错误或不准确
- 表述不清晰
- 有改进建议
- 希望增加新主题

请通过 [GitHub Issues](../../issues) 提交。

**Issue 模板**：
```
**类型**：错误修正/内容建议/新主题/其他

**描述**：
[详细描述]

**建议的改进**：
[你的建议]

**参考来源**：
[如果有数据来源或参考资料]
```

### 2. 分享案例

我们正在收集真实的 AI 协作实践案例。

**案例格式**：
- 背景（团队规模、技术栈）
- 挑战（遇到的问题）
- 解决方案（如何使用 AI）
- 结果（量化或质化的改进）
- 反思（经验教训）

请通过 Issue 或邮件提交。

### 3. 接受采访

如果你愿意分享你的转型故事，我们准备了 [采访问题清单](references/interview-questions.md)。

联系我们安排采访：
- 邮件：[你的邮箱]
- 或提交 Issue 标注 "采访"

### 4. 技术调研

帮助收集：
- AI 工具的最新数据和报告
- 行业调研结果
- 学术论⽂

请确保数据来源可靠，并在提交时注明来源。

### 5. 内容贡献

如果你想直接贡献内容：

1. Fork 本仓库
2. 创建你的特性分支 (`git checkout -b feature/你的特性`)
3. 提交你的更改 (`git commit -am 'Add: 你的特性'`)
4. 推送到分支 (`git push origin feature/你的特性`)
5. 创建 Pull Request

**写作规范**：
- 使用 Markdown 格式
- 每章一个文件，存放在对应目录
- 图片存放在 `assets/` 目录
- 遵循本书的语言风格（见下方）

---

## 语言风格指南

### ✅ 推荐

- 直接陈述观点
- 用具体场景代替抽象描述
- 用对比和反差制造张力
- 适当使用反问引发思考
- 用代码/数据支撑观点

### ❌ 避免

- "值得注意的是..."
- "让我们来看一下..."
- "这是一个很重要的问题..."
- 过度使用感叹号
- AI 套话和模板化表达
- 空洞的赞美

### 格式

- 使用**粗体**强调关键概念
- 使用代码块展示代码和命令
- 使用表格对比不同方案
- 使用引用块展示观点或案例

## 内容审核流程

1. **提交**：通过 Issue 或 PR 提交
2. **审核**：维护者审核内容
3. **反馈**：提出修改建议（如有）
4. **合并**：审核通过后合并
5. **致谢**：在致谢名单中添加贡献者
