## 写作规范

### 每章标准结构
```markdown
## 第X章：标题

> 一句话核心观点

### 开篇（2-3页）
具体场景引入，让读者有代入感

### 理论解析（4-6页）
核心概念、数据支撑、对比分析

### 实践工具（5-8页）★重点
可操作的框架/模板/清单

### 案例分析（3-4页）
真实场景，展示应用

### 常见误区（1-2页）
错误做法 + 正确做法对比

### 本章小结（1页）
3-5个要点回顾

### 反思练习（1页）
2-3个问题，让读者动手
```

### 语言风格
- ✅ 直接陈述观点，用具体场景代替抽象描述
- ✅ 用对比和反差制造张力
- ✅ 用代码/数据支撑观点
- ❌ 避免"值得注意的是"、"让我们来看一下"等套话
- ❌ 避免AI套话和模板化表达
- ❌ 避免空洞的赞美
