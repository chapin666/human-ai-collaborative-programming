## 关于本书

我们正在经历软件工程史上最深刻的变革。

GitHub Copilot、Cursor、Claude 等 AI 编程工具正在改变程序员的工作方式。但这本书不是要教你如何使用这些工具——而是要帮助你重新定义自己在 AI 时代的角色。

**核心命题**：当 AI 能写 90% 的代码时，程序员的价值从"写代码"转移到"定义问题、设计架构、把控质量、人机协作"。

## 书籍结构

### 上篇：认知与原理（14章）

建立正确的认知框架，理解 AI 时代的软件工程变革。

| 章节 | 标题 | 核心内容 |
|:---:|:---|:---|
| 第1章 | AI时代的软件工程变革 | 历史演进、行业数据、前沿趋势（Agent开发、Cowork模式、Harness Engineering） |
| 第2章 | 程序员价值的重新定义 | 价值金字塔、三种新角色、不可替代性分析 |
| 第3章 | 人机协作的三种模式 | 工具模式、结对模式、外包模式 |
| 第4章 | 认知分工与能力边界 | AI擅长 vs 人类擅长、灰色地带 |
| 第5章 | 跨职能协作的新形态 | 与产品、设计、测试、运维的协作 |
| 第6章 | 提示工程基础 | RICE框架、上下文工程 |
| 第7章 | 提示工程进阶 | Chain of Thought、Few-shot、多角色辩论 |
| 第8章 | AI输出的评估与验证 | 事实核查、逻辑验证、建立信任阈值 |
| 第9章 | 知识管理与沉淀 | Prompt库、决策日志、团队传承 |
| 第10章 | AI代码的功能性质量 | 边界条件、逻辑漏洞、审查升级 |
| 第11章 | AI代码的安全与合规 | 已知风险、新型风险、红线清单 |
| 第12章 | 性能与可维护性 | 性能陷阱、技术债、持续重构 |
| 第13章 | 代码所有权与伦理 | 责任界定、法律责任、标注规范 |
| 第14章 | 技能演进与职业规划 | 贬值技能、增值技能、职业路径 |

### 下篇：场景与实践（10章）

覆盖软件工程全生命周期的实战指南。

| 章节 | 场景 | 核心内容 |
|:---:|:---|:---|

| 第18章 | AI辅助运维与故障排查 | 部署配置、日志分析、根因分析 |
| 第19章 | 前端与移动开发的AI辅助 | 组件生成、响应式、交互实现 |
| 第20章 | 数据工程与AI辅助 | ETL设计、SQL优化、数据质量 |
| 第21章 | 算法工程与AI辅助 | 论文实现、模型服务化、实验管理 |
| 第22章 | AI辅助写作与文档 | 文章/书籍/文档的AI协作流程 |
| 第23章 | AI辅助学习新技术 | 认知地图、深度学习、项目实践 |
| 第24章 | 团队AI协作流程建设 | 规范制定、工具配置、效能度量 |

| 章节 | 标题 | 核心内容 |
